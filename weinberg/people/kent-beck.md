Kent Beck's Extreme Programming (1999) operationalized several ideas that Weinberg had articulated decades earlier. [[egoless-programming]] (1971) → collective code ownership and pair programming (XP, 1999). Weinberg's argument that code quality improves when programmers welcome review became, in Beck's hands, a concrete engineering practice. While Beck doesn't cite Weinberg as a primary influence, the conceptual lineage from "ego-free code" to "everyone owns all the code" is clear.
